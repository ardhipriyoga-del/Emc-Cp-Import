1. Riwayat Pasien Pulang Berdasarkan No. IPK

Saat pasien sudah tidak terdapat pada file import rawat inap terbaru, aplikasi menganggap pasien telah pulang.

Namun sebelum dipindahkan ke Riwayat, lakukan proses berikut:

* Simpan seluruh data operan pasien.
* Simpan seluruh pending.
* Simpan seluruh log penyelesaian.
* Simpan seluruh komentar.
* Simpan seluruh aktivitas operan.

Semua data tersebut harus diarsipkan berdasarkan:

* No. RM
* No. IPK (Episode Rawat Inap)

Bukan hanya berdasarkan No. RM.

Fitur Backup & Restore Menggunakan File Excel

Buat sistem Backup dan Restore aplikasi yang menggunakan 1 file Excel (.xlsx) sebagai media penyimpanan seluruh data aplikasi.

Backup

Saat pengguna memilih menu Backup Data, aplikasi harus membuat satu file Excel yang berisi seluruh data aplikasi dalam beberapa worksheet, antara lain:

* Pengguna (Username, Nama, Role, Password yang sudah di-hash, Status Aktif).
* Pengaturan Aplikasi.
* Logo Rumah Sakit (disimpan dalam format Base64 atau sheet khusus).
* Informasi Rumah Sakit.
* Data Pasien Aktif.
* Data Pasien Pulang.
* Data Pending Operan.
* Data Just Info.
* Riwayat Operan.
* Riwayat Pending.
* Riwayat Just Info.
* Log Aktivitas.
* Master Referensi bila ada.
* Seluruh data lain yang tersimpan pada aplikasi.

Setiap worksheet harus memiliki nama yang jelas dan struktur kolom yang konsisten agar dapat dipulihkan kembali tanpa kehilangan data.

Tambahkan worksheet AppInfo yang berisi:

* Nama aplikasi.
* Versi aplikasi.
* Versi struktur database.
* Tanggal Backup.
* Nama Rumah Sakit.
* Jumlah record setiap worksheet.
* Checksum/Signature untuk validasi file backup.

Restore

Buat menu Restore Data yang hanya menerima file Excel hasil backup aplikasi.

Saat restore dilakukan:

1. Validasi bahwa file berasal dari aplikasi EMC Admission Operan.
2. Validasi versi backup.
3. Tampilkan ringkasan jumlah data yang akan dipulihkan.
4. Berikan konfirmasi sebelum proses restore.
5. Restore seluruh data termasuk user, pengaturan, logo, pasien, pending, just info, riwayat, dan seluruh konfigurasi aplikasi.
6. Setelah restore selesai, aplikasi otomatis memuat ulang seluruh data sehingga kondisinya sama persis seperti saat backup dibuat.

Ketentuan

* File backup wajib berformat .xlsx.
* Tidak menggunakan database eksternal.
* Backup dapat dipindahkan ke komputer lain.
* Restore menghasilkan aplikasi dengan kondisi 100% sama seperti komputer asal.
* Mendukung backup dan restore secara offline tanpa koneksi internet.
* Backup harus kompatibel dengan versi aplikasi yang sama atau versi yang lebih baru yang masih mendukung migrasi data.
* Jika ditemukan perbedaan versi struktur data, aplikasi harus menjalankan proses migrasi otomatis tanpa menghilangkan data pengguna.

* Backup Otomatis: setiap selesai operan shift atau saat aplikasi ditutup, sistem otomatis membuat file backup Excel dengan nama seperti EMC_Operan_Backup_2026-07-16_0700.xlsx. Dengan begitu, jika komputer rusak atau data hilang, cukup lakukan Restore menggunakan file Excel terakhir.
